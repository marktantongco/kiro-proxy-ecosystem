🚀 Complete Step-by-Step Replication Walkthrough

Follow this structured, sequential operational process to replicate the complete self-healing double-proxy wrapper stack on any workstation from scratch.

1

Deploy Upstream Routing Core

Files Used: Clash / Mihomo Core daemon
Purpose: Sets up the foundational weighted proxy routing base on Port 7890. All external connections flow through this layer for geo-routing bypasses.
Action: Ensure Clash/Mihomo is active and listening locally: systemctl --user status mihomo or run manual binary check.

2

Execute Core Workspace Provisioner

Files Used: installers/install_owl_agent.sh
Purpose: Deduplicated Core Setup. Creates workspace directories, provisions local dynamic libraries, registers dependencies, downloads the Kiro CLI client native binaries, and creates local SQL database credentials mappings.
Action: Run: chmod +x installers/install_owl_agent.sh && ./installers/install_owl_agent.sh.

3

Execute Kiro Gateway Provisioner

Files Used: installers/install_kiro_owl_agent.sh
Purpose: Automates Python virtual environment creation, clones the kiro-gateway translation engine, and automatically updates key schema entries (such as heavy/light templates) in your local opencode.jsonc config.
Action: Run: chmod +x installers/install_kiro_owl_agent.sh && ./installers/install_kiro_owl_agent.sh.

📥 Download

4

Deploy and Start systemd Daemons

Files Used: systemd/kiro-gateway.service, systemd/owl-forward-proxy.service
Purpose: Registers the stack as automatic background processes managed by systemd under user-level context. Ensures self-healing restarts on failure.
Action: Copy to systemd directories: cp systemd/*.service ~/.config/systemd/user/, reload systemd, and enable/start the daemons.

📥 Proxy service 📥 Gateway service

5

Inject Upstream Routing Rules

Files Used: scripts/patch_owl_proxy.sh, scripts/validate_ecosystem.sh
Purpose: Overwrites downstream proxy code blocks with Clash upstream routing parameters, terminates conflicting background processes, and unsets manual variable pollution.
Action: Run: chmod +x scripts/patch_owl_proxy.sh && ./scripts/patch_owl_proxy.sh.

📥 Download Patch 📥 Download Validator

6

Secure Command Shell Launcher Wrappers

Files Used: scripts/hermes_wrapper.sh, scripts/agy_wrapper.sh, scripts/kiro_gateway_wrapper.sh
Purpose: Wraps execution of hermes, agy, and kiro-gateway inside isolated, secure, zero-leakage proxy shell blocks. Deploys launchers directly to ~/.local/bin/ for seamless PATH discovery.
Action: Copy wrappers to local path and make them executable.

📥 Hermes 📥 Antigravity 📥 Gateway

7

Verify System Integration & Direct Bypass Routing

Files Used: scripts/diagnose_opencode.sh
Purpose: The final validation gate. Performs port status analysis, audits system environment parameters, detects Unix domain socket session health, and validates HTTPS direct connection bypass rules for NVIDIA NIM and OpenCode Zen model endpoints.
Action: Run the diagnostic suite: chmod +x scripts/diagnose_opencode.sh && ./scripts/diagnose_opencode.sh. Alternatively, use the owl-check shell alias.

🛠️ Logical Workflow Sequence for Replication

1

Configure Clash Core Upstream

Establish and run mihomo or Clash-based services bound locally to **port 7890**. This processes secure, geo-routed communication tunnels.

2

Deploy Unified Agent Installers

Run the combo agent installer to deploy credentials and core components: install_kiro_owl_agent.sh or owl_agent_installer.sh. These scripts download, verify, and automatically compile Python environments.

3

Configure systemd Service Units

Deploy the kiro-gateway.service and owl-forward-proxy.service unit mappings to ~/.config/systemd/user/ and start the services.

4

Execute Environment Self-Healing

Run ./scripts/validate_ecosystem.sh to triage environment conflicts, terminate duplicate processes, and secure pings.
